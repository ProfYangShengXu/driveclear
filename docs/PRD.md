## 1. 产品定位与目标

### 1.1 产品愿景

构建一套面向行车视频的**多退化联合增强与复原流水线**，能够同时处理雾霾（haze）、眩光（glare）和低光照（low-light）三种常见的行车环境退化，输出清晰、细节丰富、色彩自然的增强视频，辅助驾驶安全感知与后续视觉任务。

### 1.2 核心问题

| 退化类型 | 现有方法局限 | 本方案方向 |
|----------|-------------|-----------|
| 雾霾 | DCP 单帧去雾无法利用时序信息，雾重区域色彩失真 | 时域雾层估计 + DCP 自适应融合 |
| 眩光 | 传统方法难以去除高亮光晕/镜头 flare，易产生黑色伪影 | 亮通道检测 + 局部亮度抑制 + 时间域插值 |
| 低光照 | CLAHE/Retinex 单一增强过度放大噪声 | 自适应伽马 + 亮度通道 CLAHE + 运动蒙版增强 |
| **混合退化** | 现有方法均只能处理单一类型，无法应对同时存在多种退化的场景 | 模块化流水线，按帧自适应调度各模块 |

### 1.3 核心创新点

1. **前景/背景解耦的轻量多帧融合去雾**：运动区域（车辆/行人）采用时域减法保持边缘锐度，静止区域（天空/背景）采用 DCP 保证色彩自然
2. **运动蒙版增强**：增强增量仅作用于亮度通道的检测到的运动区域，避免天空/背景过增强
3. **联合退化流水线架构**：各退化处理模块可插拔组合，适应环境快速变化

## 3. 用户场景

### 场景 A：雾天高速公路行车

```
输入：浓雾中高速公路行车视频（前方车辆轮廓模糊）
期望输出：前方车辆轮廓清晰、车道线可辨、天空区域色彩自然
关键指标：SSIM ↑, NIQE ↓
```

### 场景 B：夜间城市道路眩光

```
输入：对向车灯/路灯产生的镜头眩光 + 低光照
期望输出：眩光区域被有效抑制、路面和行人细节增强
关键指标：BRISQUE ↓, 主观眩光可见度评分
```

### 场景 C：黄昏/隧道出口混合退化

```
输入：雾霾 + 低光照 + 隧道出口过曝 三者同时存在
期望输出：三个退化同时被处理，无模块间冲突伪影
关键指标：PSNR ↑, 消融实验量化各模块贡献
```

## 6. 评估指标与验收标准

### 6.1 定量指标

| 指标 | 类型 | 目标方向 | 说明 |
|------|------|---------|------|
| PSNR ↑ | 全参考 | ≥ 基线 DCP | 峰值信噪比，越高越好 |
| SSIM ↑ | 全参考 | ≥ 基线 DCP | 结构相似性，越高越好 |
| NIQE ↓ | 无参考 | < 原始帧 | 自然图像质量评估器，越低越好 |
| BRISQUE ↓ | 无参考 | < 原始帧 | 盲参考图像空间质量评估，越低越好 |
| 处理速度 | 性能 | ≥ 5 FPS | 720p 分辨率下流水线吞吐 |

## 8. 开放问题与风险

### 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 眩光去除效果不佳，/产生黑色伪影 | 中 | 高 | 备选方案：inpainting 补全 + 与 DCP 融合时的色彩校正 |
| 多模块级联导致处理速度 < 5 FPS | 中 | 中 | 快速模式降采样 1/4 计算传输率；关键模块(low-light)仅在检测到低光帧时启用 |
| GLARE 数据集格式不一致/标注缺失 | 低 | 中 | 人工筛选可用片段，社交媒体视频作为补充 |
| Zero-DCE / NDELS 模型环境依赖复杂 | 中 | 低 | 独立 conda 环境；若装不上则用论文提供的在线 demo 结果作参考 |

### 范围风险

| 风险 | 说明 |
|------|------|
| 多帧对齐可能引入运动拖影 | 若模板匹配效果差，降级为单帧逐模块处理，放弃对齐环节 |
| 混合退化场景缺少 ground truth | 无参考指标（NIQE/BRISQUE）作为主要评估手段 |
| 眩光与车灯难以区分 | 若区分难度大，将策略聚焦于"抑制非自然高亮区域"，保留车灯形态 |

### 确定不做（本次项目范围外）

- **实时视频流处理**（仅处理完整视频文件）
- **GPU 加速**（纯 CPU 实现，后续可扩展）
- **端到端深度学习模型**（仅 Zero-DCE / NDELS 作为对比基线，核心方法为传统图像处理）
- **移动端/嵌入式部署**
